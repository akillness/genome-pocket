# 05 · 지원 크레딧 & OpenAI API 실험 설계

> 워크숍 운영진 추가 안내(참가자 1인당 Codex Credit $100 + OpenAI API $50) 반영.
> 개발자(정장영) 관점에서 **"50달러로 무엇을 검증하고 무엇을 포기할지"** 를 미리 정해
> 3시간 안에 의미 있는 프로토타입(서울환경연합 언론보도 도구)으로 잇는 계획.
> 출처 검증: OpenAI 공식 문서 https://developers.openai.com/api/docs/ (API 목록 직접 대조 확인).

## 0. 지원 내용 요약 (확정 사실)

| 항목 | 금액 | 용도 | 단위 |
|------|------|------|------|
| Codex Credit | $100 | Codex 사용(코드 생성·실행) | **참가자 1인당** |
| OpenAI API | $50 | Codex 내/외부에서 OpenAI API 직접 호출 실험 | **참가자 1인당** |

- 팀 단위가 아니라 **1인당** 지원 → 한 팀(개발자+활동가)이면 합산 활용 가능.
- 제공 방식·사용 시작법은 운영진이 별도 재안내 예정(워크숍 전 확인 필요).

## 1. 이번 워크숍에서 테스트 가능한 OpenAI API (공식 문서 대조 완료)

| API | 핵심 기능 | 우리 프로젝트(언론보도 도구) 연결 포인트 |
|-----|-----------|--------------------------------------------|
| **Responses API** | 텍스트 생성·추론·이미지 입력·파일/웹 검색·함수 호출(최신 기본 API) | 기사 본문 요약·분류·정성평가의 1순위 진입점. 함수 호출로 구조화 출력 |
| **Chat Completions API** | 대화형 응답(기존 방식) | 익숙한 패턴이 필요할 때. 신규는 Responses 권장 |
| **Images API** | 이미지 생성·편집 | 우선순위 낮음(보도 도구엔 부차적). 캠페인 카드 등 부가 실험용 |
| **Audio API** | 음성 인식·번역·음성 합성 | 회의록/인터뷰 음성→텍스트 후 분류 파이프 연결 가능 |
| **Realtime API** | 실시간 음성·대화 인터랙션 | 데모 임팩트는 크나 비용·복잡도 높음 → 워크숍선 보류 권장 |
| **Embeddings / Vector Stores / File Search** | 유사도 검색·RAG | **핵심**: 과거 보도 아카이브 검색·중복 기사 군집·"같은 사안" 묶기 |
| **Moderation API** | 입출력 안전성 점검 | 무료/저비용. 외부 공개 데모 시 가드레일 |
| **Batch / Evals / Fine-tuning** | 대량처리·평가·모델 조정 | Evals로 분류 프롬프트 품질 비교. Batch로 과거 기사 대량 일괄 분류(50% 저렴) |

> 참고 문서: [API 문서 허브](https://developers.openai.com/api/docs/) · [Responses 개요](https://developers.openai.com/api/reference/responses/overview) · [Chat Completions 개요](https://developers.openai.com/api/reference/chat-completions/overview)

## 2. $50로 무엇이 되고 무엇이 안 되나

### 충분히 가능 (워크숍 목표)
- 현장 문제 기반 **AI 기능 프로토타입** 1~2개 (예: 기사 자동 분류기)
- 설문·문서·회의록 **요약/분류 소형 워크플로우**
- **File Search/임베딩**으로 내부 자료 검색 데모
- 이미지·음성 입력 활용 **간단한 UX 실험**
- **Evals/프롬프트 비교**로 결과 품질 점검

### 제한적·부적합 (이번 크레딧으로는 X)
- 실서비스급 대규모 트래픽 처리
- 장시간·대용량 음성/영상 처리
- 다수 동시 사용자 상용 제품 운영
- 장기 데이터 파이프라인/자동화 상시 운영

> 핵심 메시지: 50달러는 **"운영"이 아니라 "감각 검증(feasibility probe)"용**.
> "무엇이 가능한가 / 어디가 비용·속도·데이터 구조상 한계인가 / Codex 결과물을 API 프로토타입으로 어떻게 잇는가"를 체감하는 것이 목표.

## 3. 비용 가드레일 (50달러를 태우지 않기)

- **모델 선택**: 분류·요약 같은 정형 작업은 소형 모델(미니급) 우선, 어려운 정성평가만 상위 모델로 에스컬레이션.
- **대량 작업은 Batch API**: 과거 기사 백필(backfill) 분류는 Batch로 약 50% 절감 + 동시성 한도 회피.
- **임베딩 캐싱**: 같은 기사 재임베딩 금지 — 해시 키로 Vector Store에 1회만 적재.
- **컨텍스트 다이어트**: 기사 전문 대신 제목+리드+핵심 단락만 투입(토큰·정확도 동시 개선).
- **Moderation은 저비용**: 외부 데모 전 입출력 점검에 부담 없이 사용.
- **Realtime/대용량 오디오 보류**: 단가·시간 소모가 커 워크숍 예산에 부적합.
- **예산 알람**: OpenAI 대시보드에서 사용량 한도(usage limit) 설정 후 시작.

## 4. 3시간 안의 API 실험 플랜 (03-tech-challenges-3h.md와 연계)

| 단계 | 실험 | 사용 API | 산출물 / 검증 질문 |
|------|------|----------|--------------------|
| 1 | 기사 1건 자동 요약·핵심추출 | Responses | "본문→구조화(제목/매체/요지/논조) 가능한가?" |
| 2 | 보도 정성 라벨링(단순복사 vs 심층) | Responses + 함수호출 | "라벨 일관성이 나오나? 프롬프트로 통제되나?" |
| 3 | 과거 보도 검색 데모 | Embeddings + Vector Store + File Search | "같은 사안 기사 묶기·중복 제거가 되나?" |
| 4 | 분류 품질 비교 | Evals | "프롬프트 A vs B 정확도 차이 측정" |
| 5 | (여유 시) 대량 백필 | Batch | "과거 기사 일괄 분류 비용·시간 추정" |

> 활동가 사용 표면은 여전히 구글 생태계(시트/Looker Studio)로 유지하고,
> 분류·정성평가·검색 모듈만 OpenAI API로 얹는 02/03 문서의 아키텍처를 그대로 따른다.

## 5. Codex → API 프로토타입 잇기 (이번 지원의 진짜 의도)

1. **Codex($100)**: 문제를 구조화하고 스캐폴딩(수집기·시트 연동·분류 함수 골격)을 빠르게 생성.
2. **OpenAI API($50)**: 그 골격의 "지능" 부분(요약·분류·검색)을 실제 호출로 채워 **동작하는 프로토타입**으로 검증.
3. **결과 해석**: 워크숍 종료 시 "어디까지 됐고, 운영하려면 무엇(비용/데이터/인프라)이 더 필요한가"를 활동가가 의사결정할 수 있는 형태로 정리.
