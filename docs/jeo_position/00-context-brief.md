## 2. 사용자 페르소나

### Primary — 현장 활동가 (대시보드 사용자)
- 기획·현장 활동·캠페인으로 업무 과중. 성과 평가 시점에 수많은 언론보도를 직접 모니터링·취합할 여력이 없다.
- 기술 수준: 앱스스크립트 **메뉴 버튼 클릭 수준**까지 사용 가능. 코드 작성/디버깅은 기대하기 어렵다.
- 니즈: "버튼 누르면 보이는" 접근성 높은 도구. 검색·복붙·수기 정리에서 해방.

### Secondary — 사무처장(이동이) (의사결정자)
- 조직 운영 총괄, 업무 자동화 도입·비전 수립·성과 평가 체계 구축을 리딩.
- 니즈: 사업 평가·캠페인 방향·미디어 소통 전략에 쓸 **신뢰할 수 있는 성과 데이터**.

## 3. Jobs To Be Done (JTBD)

> *"When ___ , I want to ___ , so I can ___ ."* (Moesta/Christensen)

- **JTBD-1 (성과 증명)**: *사업 평가 시점이 오면*, 우리 단체가 만든 언론보도를 빠짐없이 모아 정리하고 싶다, *그래서 세운 목표 대비 정량·정성 성과를 증명할 수 있도록.*
- **JTBD-2 (의제 추적)**: *캠페인을 진행하는 동안*, 어떤 언론사·기자가 우리 의제에 관심을 갖는지 알고 싶다, *그래서 미디어 소통 전략을 세울 수 있도록.*
- **JTBD-3 (반복업무 제거)**: *매번 기사를 검색·수기 아카이빙해야 할 때*, 그 작업을 자동화하고 싶다, *그래서 활동가가 본연의 캠페인 업무에 집중할 수 있도록.*

핵심 통찰: 단체는 이미 "언론보도 횟수"를 KPI로 설정했지만, **데이터가 휘발되어 KPI를 증명하지 못하는 것**이 진짜 고통이다.

## 4. North Star Metric (가설)

> pm-skills의 *North Star Framework* — 전달된 고객 가치를 가장 잘 포착하는 단일 지표 + 선행지표.

- **North Star (제안)**: "활동가의 수기 개입 없이 자동으로 분류·아카이빙된 **유효 보도 건수 / 월**."
  - 단순 "수집 건수"가 아니라 "사업/팀에 매핑되고 품질 평가까지 끝난 건수"여야 가치와 직결된다.
- **선행지표 (Input metrics)**
  1. 키워드·소스 커버리지(놓친 기사 비율 ↓)
  2. 자동 분류 정확도(팀·사업 매핑 정확도)
  3. 심층/단순(보도자료 복사) 기사 분류 정확도
  4. 활동가 1인당 월 절감 시간
  5. 언론사·기자 아카이브 완성도

## 5. 가정 지도 (Assumption Map) — 무엇부터 검증할까

> 2×2 (중요도 × 모름의 정도). 오른쪽 위(중요한데 모르는 것)부터 워크숍에서 검증.

| 가정 | 중요도 | 확신도 | 워크숍 검증 방법 |
|------|:---:|:---:|------|
| 구글 알리미/잔디 데이터에 충분한 메타데이터(언론사·기자·날짜·URL)가 남는다 | 高 | 낮음 | 실제 잔디 export 샘플 확인 |
| "사업계획서 ↔ 기사" 분류 기준이 문서로 존재하거나 만들 수 있다 | 高 | 낮음 | 사업계획서/팀 구조 샘플 확인 |
| "보도자료 복사 vs 심층 기사"를 LLM이 판별할 만한 신호가 본문에 있다 | 高 | 중간 | 라벨링된 기사 5~10건으로 즉석 평가 |
| 활동가가 대시보드를 버튼/링크 수준으로 쓸 수 있다 | 高 | 높음 | 25분 싱크에서 사용 흐름 합의 |
| 키워드 정확도 문제는 규칙+LLM 보정으로 해결 가능하다 | 中 | 중간 | 키워드 오탐 사례 수집 |
