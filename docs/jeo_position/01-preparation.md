# 01 · 개발자 준비 체크리스트 (Preparation)

> pm-skills의 *Continuous Discovery / Interview Guide* 관점. 워크숍 3시간을 코딩에 쓰려면, 발견(질문)은 사전에 끝내 둔다.

## A. 워크숍 전 (사전, 비동기)

### A-1. 데이터 형태 사전 확인 (가장 중요)
사전 문서는 "데이터 바로 가져올 수 있음 / 원본 공유 가능"이라고 명시했다. 당일에 데이터 형태를 처음 보면 3시간이 날아간다.

- [ ] **잔디 채널 export 샘플** 1개월치 요청 — 한 건당 어떤 필드가 남는가? (제목/URL/날짜/언론사/요약/본문 유무)
- [ ] **사업계획서 / 팀·사업 구조** 샘플 요청 — 분류 기준(팀명, 세부 사업명, 키워드)이 어떤 형태로 존재하는가?
- [ ] **라벨링된 기사 예시** 요청 — "보도자료 복사 기사" 3건 + "심층 기사(활동가 코멘트·추가 취재 포함)" 3건. 정성평가 기준의 그라운드 트루스.
- [ ] **현재 키워드 목록** — 서울환경연합, 서울환경운동연합 등 + 오탐(동음이의·무관 기사) 사례.
- [ ] **공개 범위 / 보안** — 원본 기사 본문 저장 가능 여부, 외부 LLM API로 본문 전송 가능 여부(저작권·개인정보).

### A-2. 개발 환경 사전 셋업 (당일 셋업 시간 0으로)
- [ ] 구글 계정·권한: Apps Script, Google Sheets API, (선택) Google Drive API 활성화. 활동가가 이미 쓰는 **구글 생태계 안에 결과물을 두는 것**이 인수인계에 유리하다.
- [ ] LLM API 키 준비 (분류·정성평가용). 키워드 정확도/심층 판별은 규칙만으로 한계 → LLM 보정 전제.
- [ ] 로컬 프로토타이핑 스택 준비: Python(수집·파싱·LLM 호출) + 경량 대시보드(Streamlit/Looker Studio/구글시트). 강점인 RAG·API·Bot 경험 재활용.
- [ ] 샘플 데이터로 **수집→파싱→1건 분류** 파이프라인을 미리 한 번 돌려 본다(스파이크). 막히는 지점을 당일이 아닌 사전에 발견.

### A-3. 기대치 정렬
- [ ] 사전 제출안과 다른 방향이면 **반드시 사전 협의** (문서 명시 사항).
- [ ] "함께 방법을 찾고 쉽게 설명" 기대 → 당일 페어링/화면공유 + 용어 풀어 설명 모드로 진행.

## B. 25분 싱크 미팅 어젠다 (당일 시작 전)

문서가 권장한 5단계 + 개발자가 던질 핵심 질문.

1. **현장 문제 듣기** — 성과 평가 시 가장 답답했던 최근 사례 하나를 구체적으로.
2. **배경·제약·우선순위** — 사업 평가 주기? 한 번 평가에 기사 몇 건? 누가 정리하나?
3. **사전 질문 확인** (운영진 초안 + 개발자 추가):
   - 우선 대상 뉴스 소스·키워드는? (구글 알리미만? 포털·언론사 직접?)
   - 사업계획서 ↔ 기사 분류 기준의 형태는? (팀/세부사업 트리가 있나)
   - "보도자료 복사" vs "심층 기사"를 가르는 **현장의 판단 기준**은? (활동가 코멘트 인용? 기자 추가 취재? 단독?)
   - 키워드 오탐 사례 — 어떤 무관 기사가 자꾸 걸리나?
4. **당일 데이터/샘플 형태·공개 범위 최종 확인.**
5. **3시간 안에 만들 "가장 작은 결과물(MVP)" 합의** → [03-tech-challenges-3h.md](03-tech-challenges-3h.md)의 시나리오 중 택1.

## C. 당일 운영 원칙

- **Observe → Act → Observe**: 데이터를 먼저 보고(샘플 10건), 한 단계 만들고, 결과를 활동가와 같이 본다. 큰 설계보다 빠른 루프.
- **인수인계 가능성 우선**: 활동가가 버튼 수준으로 쓸 수 있어야 한다. 화려한 스택보다 **구글시트/Apps Script/Looker Studio처럼 단체가 유지할 수 있는 표면**을 1순위 후보로.
- **데모 가능한 한 조각**: 3시간 끝에 "키워드→기사→자동 분류→대시보드 카드 1줄"이 끝까지 흐르는 얇은 수직 슬라이스를 목표로.
- **남길 산출물**: 코드 + 짧은 운영 가이드(키 갱신·키워드 추가 방법) + 다음 단계 백로그.
